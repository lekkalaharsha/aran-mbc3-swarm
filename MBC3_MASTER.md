## 4. AERIS-10 RADAR DRIVER

### What it is
AERIS-10 = FMCW phased-array radar, 10.5 GHz, ADAR1000 beamsteering chip.
Open-source project (user is a contributor).
In this project: used for target detection in 6 sectors (panels A–F, 60° each).

### ROS2 Package: `~/aran_mbc/aeris10_driver/`
```
aeris10_driver/
├── aeris10_driver/
│   ├── aeris10_usb.py    — USB binary frame parser
│   └── driver_node.py    — ROS2 node, publishes 6× PointCloud2
├── config/aeris10_driver.yaml
├── launch/aeris10_driver.launch.py
├── setup.py / package.xml
```

### Topics published
```
/radar_A/scan/points  (PointCloud2, panel 0°,   sector [-30°, +30°])
/radar_B/scan/points  (PointCloud2, panel 60°,  sector [+30°, +90°])
/radar_C/scan/points  (PointCloud2, panel 120°, sector [+90°,+150°])
/radar_D/scan/points  (PointCloud2, panel 180°, sector [+150°,+210°])
/radar_E/scan/points  (PointCloud2, panel 240°, sector [+210°,+270°])
/radar_F/scan/points  (PointCloud2, panel 300°, sector [+270°,+330°])
```
These feed directly into `radar_fusion/detection_node`.

### Run in simulation mode (no hardware)
```bash
ros2 launch aeris10_driver aeris10_driver.launch.py sim_mode:=true
```

### USB Protocol (adapt when hardware arrives)
Frame format (little-endian binary):
```
Header 16 bytes:  magic[2]=0xAE10 | frame_id[4] | timestamp_us[8] | n_returns[2]
Per return 16 bytes: range_m[f32] | az_deg[f32] | el_deg[f32] | power_dBm[f32]
```
Update `AERIS10_VID` / `AERIS10_PID` in `aeris10_usb.py` from `lsusb` output.

### 5.2 Motor Selection
**Recommended: T-Motor Antigravity MN3110-26 (700KV)**
- Stator: 31×10 mm | Weight: 115 g | Shaft: 4 mm
- Mount: **4× M3, 19 mm bolt-circle diameter**
- With 12×4.5 on 6S: ~2000–2200 g thrust per motor

**Backup: EMAX MT3510+ (600KV)** — 16 mm bolt-circle, M3

**Thrust calculation:**
```
Hover thrust needed = 5.83 × 9.81 / 6 = 9.53 N = 972 g per motor
Max thrust per motor ≈ 2100 g
TWR = 2100/972 = 2.16 ✓
Hover throttle = √(972/2100) = 68%
```
